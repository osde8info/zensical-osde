[dot.mobi](http://mtld.mobi/) web pages have some **special** conventions, standards and styles (some of them mandatory) that might be **different** to other web standards and conventions you find on the rest of the web:

- [http://dev.mobi/](http://dev.mobi/)
- [http://dev.mobi/styleguides](http://dev.mobi/styleguides)
- [http://dev.mobi/files/dotMobi\_Domain\_Naming\_Guidelines\_1.0.html](http://dev.mobi/files/dotMobi_Domain_Naming_Guidelines_1.0.html)
- [http://dev.mobi/files/dotmobi\_Switch\_On\_Web\_Developer\_Guide.html](http://dev.mobi/files/dotmobi_Switch_On_Web_Developer_Guide.html)

for example:  
URLS:

> _DotMobi rules stipulate that there should be a site at the second level domain. This means that mobile users can enter the URL as example.mobi instead of www.example.mobi, thereby saving keystrokes_.  

I18N:  

> _Identify national variations of dotMobi sites by using the corresponding country code top level domain identifier (ccTLD) as the third level domain identifier._ie uk.mydomain.mobi !

MIME type:

> _For XHTML-MP, the recommended MIME type is `application/vnd.wap.xhtml+xml` or `application/xhtml+xml`. Unlike HTML, XHTML-MP should not be served as `text/html`._  

  
You can develop, test and validate your dot mobi sites for compliance by using:

- [http://dev.mobi/topic/tools](http://dev.mobi/topic/tools)
