## [Conary](http://wiki.rpath.com/wiki/Conary:Overview) Whitepapers

Conary replaces existing software package management solutions (such as RPM and dpkg) with one designed to enable loose collaboration across the Internet. Unlike traditional packages, Conary packages are defined by components and files stored in networked repositories.

- [An Introduction to the Conary Software Provisioning System](http://www.rpath.com/technology/conary.html "http://www.rpath.com/technology/conary.html"): A broad overview of Conary's goals; read this first to understand what problems Conary is designed to solve ([PDF](http://www.rpath.com/technology/conary.pdf "http://www.rpath.com/technology/conary.pdf")).

- [Repository-based System Management Using Conary](http://www.rpath.com/technology/techoverview/ "http://www.rpath.com/technology/techoverview/"): An overview of the technology and high-level design of Conary ([PDF](http://www.rpath.com/technology/techoverview.pdf "http://www.rpath.com/technology/techoverview.pdf")).

- [Building Linux Software with Conary](http://www.rpath.com/technology/building/ "http://www.rpath.com/technology/building/"): This paper describes best practices in Conary packaging
