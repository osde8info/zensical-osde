I've often heard that "The Code is the Documentation" but how about "The Documentation is the Code" ?

[AppJet](http://appjet.com/) have some amazing [documentation](http://appjet.com/docs/) where "the documentation really is the application" since underneath most [examples](http://appjet.com/docs/devguide/intro) there is a button that says "[new app from this](http://appjet.com/app/167711168/ide)" that creates an app from the code directly in the online [AppJet IDE](http://appjet.com/app/167711168/ide) !
